# Refactor Execution Plan (Modular Monolith)

This plan keeps the current deployment model (single backend process) while moving the codebase to clean modular boundaries.

## Completed baseline

- Unified API registration entry: `app/interfaces/api/v1/router_registry.py`
- New orchestration layer: `app/application/workflows/*`
- New domain layer structure: `app/domains/*`
- New infrastructure layer structure: `app/infrastructure/*`
- Shared contracts: `app/shared/contracts.py`
- `main.py` now registers routers via `include_v1_routers(app)`

## Current migration scope

- Task, batch, archive, QA, and evaluation routers are routed through workflows.
- `app/api/routes.py` is now a compatibility shell to avoid breaking legacy imports/tests.
- Frontend route imports are grouped by feature entry modules.

## Next steps

- Normalize OCR/LLM provider boundary and error semantics.
- Continue moving remaining legacy logic into workflow/domain layers.
- Add stronger architecture guard tests and remove duplicated legacy paths.
